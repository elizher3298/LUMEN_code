# LUMEN

LUMEN is a shallow multilayer perceptron for daily solar radiation (SRAD)
estimation, designed for low-cost edge deployment in agricultural IoT settings.
It fuses Himawari satellite and AgERA5 reanalysis inputs through domain-informed
feature engineering (clearness-index normalization and solar angle). The model
is trained on Korea/Japan ground stations and validated independently on PANGAEA
stations.

This repository contains the data-preparation and training/validation pipeline
used to produce the results reported in the paper.

## Repository structure

```text
.
├── prepare_lumen_data.py     # Step 1: build experiment data from raw CSVs
├── main.py                   # Step 2: cross-validation training + validation
├── src/
│   ├── model.py              # ModelConfig, ModelArchitectures, model builder
│   ├── trainer.py            # StationBasedCVTrainer (station-based 5-fold CV)
│   ├── validator.py          # ModelValidator (independent-set evaluation)
│   ├── feature.py            # FeatureEngineer (feature/target construction)
│   ├── data_loader.py        # DataLoader
│   └── experiment_manager.py # ExperimentManager
├── configs/
│   ├── gpu_experiment_plan.csv   # Experiment definitions (see "Configuration")
│   └── feature.yaml              # Feature/target definitions
├── data/                     # Raw input CSVs go here (see "Data")
├── experiments/              # Generated by prepare_lumen_data.py
└── results/                  # Generated by main.py at run time
```

All paths resolve relative to the file locations, so the pipeline runs from a
fresh clone without editing any paths.

## Data

Place the raw input CSVs in `data/`:

| File | Used by | Required |
| --- | --- | --- |
| `training_korjap.csv` | `prepare_lumen_data.py` | Yes |
| `validation_korjap.csv` | `prepare_lumen_data.py` | Optional |
| `validation_pangaea.csv` | `prepare_lumen_data.py` | Optional |
| `station_info.csv` | `main.py` (validation) | Optional |

`prepare_lumen_data.py` reads these raw files, applies feature engineering, and
writes the per-experiment files into `experiments/<experiment_id>/`
(`train_data.csv`, `val_korjap_data.csv`, `val_pangaea_data.csv`). You do not
place anything in `experiments/` by hand.

## Configuration

Experiments are defined in `configs/gpu_experiment_plan.csv`. Each row provides:

- `id`: experiment ID (e.g., `exp5`)
- `features`: space-separated feature column names
- `target`: target column type (default `srad`)
- `normalization`: normalization scheme (default `standard`)

Feature and target construction is defined in `configs/feature.yaml`.

## Usage

### Step 1 — Prepare experiment data

```bash
python prepare_lumen_data.py exp5
```

Optional station filtering:

```bash
python prepare_lumen_data.py exp5 --min-station-id 4001
```

This creates `experiments/exp5/` with the processed train/validation CSVs.

### Step 2 — Train and validate

Run every architecture x dropout combination enabled in
`src/model.py` (`ModelArchitectures`):

```bash
python main.py exp5
```

Run a single architecture:

```bash
python main.py exp5 --architecture <name> --dropout-rate 0.1
```

`<name>` must be an entry in `ModelArchitectures.ARCHITECTURES` (see
`src/model.py`).

### `main.py` options

| Option | Default | Description |
| --- | --- | --- |
| `experiment_id` | (required) | Experiment ID, e.g. `exp5` |
| `--base-dir` | repo root | Base directory holding `configs/`, `experiments/`, `data/`, `results/` |
| `--output-dir` | `<base-dir>/results/<experiment_id>` | Output directory |
| `--architecture` | none (runs all combinations) | Single architecture name |
| `--dropout-rate` | `0.2` | Dropout rate (only with `--architecture`) |
| `--use-batch-norm` | off | Enable batch normalization |
| `--epochs` | `100` | Maximum epochs |
| `--batch-size` | `32` | Batch size |
| `--patience` | `20` | Early-stopping patience |
| `--cv-threshold` | `90.0` | CV_RMSE (%) rejection threshold |

## Pipeline

`main.py` runs the following steps for each combination:

1. Read the experiment configuration from `configs/gpu_experiment_plan.csv`.
2. Load `experiments/<id>/train_data.csv` and split into features, target, and metadata.
3. Build the model and run station-based 5-fold cross-validation.
4. Select the best fold by validation mRMSE (folds above `--cv-threshold` are rejected).
5. Evaluate the best-fold model on the KorJap and PANGAEA sets when present.
6. Write `final_results.json` and logs.

## Outputs

Written to `results/<experiment_id>/` (per combination subfolders in batch mode):

- `logs/experiment_<timestamp>.log`
- `experiment_config.json`
- `final_results.json` (includes `cv_results.best_fold`)
- `batch_summary.json` (batch mode only)

Per-fold model checkpoints and scalers are saved by the trainer under the same
output directory.
